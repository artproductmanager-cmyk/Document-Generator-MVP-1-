## ⚠️ Требует настройки для продакшена

### 1. YooKassa Integration
**Статус**: Подготовлено, требует API ключи

**Что нужно**:
- Зарегистрироваться на yookassa.ru
- Получить `shop_id` и `secret_key`
- Добавить в переменные окружения Supabase
- Раскомментировать код интеграции в `/supabase/functions/server/index.tsx`

**Файлы для изменения**:
- `/supabase/functions/server/index.tsx` (endpoint create-payment)

### 2. Cron для сброса лимитов
**Статус**: Endpoint готов, требует настройка планировщика

**Что нужно**:
- Настроить внешний cron-сервис (cron-job.org, easycron.com)
- ИЛИ GitHub Actions (пример в BACKEND_INTEGRATION.md)
- ИЛИ Supabase Cron (если доступно)

**Частота**: 1-го числа каждого месяца в 00:00

### 3. Email уведомления (опционально)
**Статус**: Не реализовано

**Варианты**:
- Supabase Email Templates
- SendGrid / Mailgun
- Resend

**Сценарии**:

- Успешная оплата
- Истечение подписки

### Тестирование функционала

#### Регистрация и вход
```bash
1. Откройте /signup
2. Заполните форму: email, пароль, имя
3. Нажмите "Зарегистрироваться"
4. Автоматический вход в систему
5. Редирект на главную
```

#### Создание документа
```bash
1. Войдите в систему
2. Выберите любой документ
3. Заполните форму
4. Нажмите "Скачать Word"
5. Документ сохранится в личном кабинете
6. Проверьте /dashboard
```

#### Проверка лимитов
```bash
1. Создайте 3 документа (free tier)
2. Попробуйте создать 4-й
3. Появится ошибка и предложение улучшить тариф
4. Перейдите на /pricing
```

#### Платежи (тестовый режим)
```bash
1. Откройте /pricing
2. Выберите тариф
3. Нажмите кнопку оплаты
4. Появится уведомление о создании платежа
5. (В продакшене — редирект на YooKassa)
```
