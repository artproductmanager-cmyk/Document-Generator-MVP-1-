## 📋 Твой план действий (15 минут):

### ШАГ 1: Проверь Supabase (1 минута)

Открой: https://supabase.com/dashboard

**Убедись что:**
- ✅ Edge Function "server" работает (зелёный статус)
- ✅ Таблица `kv_store_297fba9e` существует

**Скопируй ключи:**
- Settings → API → Project URL
- Settings → API → anon public key

Сохрани их — понадобятся!

---

### ШАГ 2: Установи Vercel CLI (30 секунд)

```bash
npm install -g vercel
```

**Проверь:**
```bash
vercel --version
```

Если не работает, используй:
```bash
npx vercel
```

---

### ШАГ 3: Залогинься в Vercel (30 секунд)

```bash
vercel login
```

Выбери Email или GitHub и подтверди в браузере.

---

### ШАГ 4: Задеплой проект (5 минут)

```bash
# Из корневой папки проекта
vercel
```

**Ответь на вопросы:**
```
? Set up and deploy? → Y
? Which scope? → [Твой аккаунт]
? Link to existing project? → N
? Project name? → dokumentator
? Directory? → ./
? Override settings? → N
```

**Результат:**
```
✅ Deployed to https://dokumentator-xxx.vercel.app
```

---

### ШАГ 5: Добавь переменные (2 минуты)

**Через CLI:**
```bash
vercel env add VITE_SUPABASE_URL
# Вставь: https://xxx.supabase.co

vercel env add VITE_SUPABASE_ANON_KEY  
# Вставь: eyJhbG...
```

**Или через Web:**
1. https://vercel.com/dashboard
2. Проект → Settings → Environment Variables
3. Добавь оба ключа

---

### ШАГ 6: Production деплой (1 минута)

```bash
vercel --prod
```

**Готово!**
```
✅ Production: https://dokumentator.vercel.app
```

---

### ШАГ 7: ТЕСТИРОВАНИЕ! (5 минут)

Открой сайт и проверь:

**1. Регистрация:**
- Нажми "Войти" → "Регистрация"
- Создай аккаунт
- ✅ Должен залогиниться

**2. Создание документа:**
- Выбери "Договор аренды"
- Заполни форму
- Скачай документ
- ✅ Должен скачаться Word файл

**3. История:**
- Открой "Личный кабинет"
- ✅ Документ в истории
- ✅ "2 документа осталось"

**4. Лимиты:**
- Создай ещё 2 документа
- Попытайся создать 4-й
- ✅ "Лимит достигнут"

**5. Pricing:**
- Открой "Тарифы"
- ✅ Синий баннер "Мягкий запуск"
- ✅ Кнопки "Скоро будет доступно"
